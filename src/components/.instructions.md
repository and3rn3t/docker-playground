# Component Instructions

## Rules

- **Do NOT modify** files in `src/components/ui/` — these are shadcn/ui primitives managed by the shadcn CLI.
- Application components live directly in `src/components/` and compose UI primitives.
- Use Phosphor Icons (`@phosphor-icons/react`), never Lucide or Heroicons.
- Use `cn()` from `@/lib/utils` to merge Tailwind classes.
- Use Framer Motion `<motion.div>` for animations; keep durations under 400ms.
- Use `sonner` for toast notifications.
- Keep components focused on a single responsibility; extract reusable logic into `src/hooks/`.

## Patterns

- Persistent state: `useKV<T>(key, defaultValue)` from `@github/spark/hooks`
- Ephemeral state: React `useState`
- All Docker state mutations must go through the command parser in `src/lib/docker-parser.ts` — never mutate containers/images directly from UI components.
- Terminal output must respect the `TerminalLine` type discriminator (`command | output | error | success`).

## Import Convention

```tsx
import { ComponentName } from '@/components/ui/component-name'
import { cn } from '@/lib/utils'
import { IconName } from '@phosphor-icons/react'
```
