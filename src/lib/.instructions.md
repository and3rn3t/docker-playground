# Core Library Instructions

## `docker-parser.ts` — The Command Parser

All state mutations flow through `parseCommand()`. See `.github/copilot-instructions.md` for the full step-by-step guide on adding new commands.

### Key Constraints

- Never allow direct state mutation outside the parser
- Validate all user input; handle edge cases (missing args, duplicate names, port conflicts)
- Keep command output formatted for monospace terminal display
- Each `handleXxx()` function receives `(parts, state, updateState)` and returns `CommandResult`

## `types.ts` — Domain Types

All shared interfaces live here. When adding new domain concepts, define them in this file — not inline in components.

## `tutorials.ts` — Tutorial Definitions

Each `TutorialStep` has `expectedCommand` (string or string[]), optional `validation` function, and progressive `hints`. See `.github/copilot-instructions.md` for full tutorial authoring guide.

## `achievements.ts` — Achievement Definitions

Achievements are unlocked by a `requirement` function that checks `AchievementCheckData`. Achievement IDs must match tutorial IDs when a tutorial completion triggers an achievement.
